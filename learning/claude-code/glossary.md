# Claude Code 术语表

> 只有当你**真正理解**一个术语后，才会加入这里。这不是词典，是压缩后的知识。

---

## 主循环 (Master Loop)

Claude Code 的核心交互模式。每次用户输入后，大模型进入一个循环：思考 → 决定是否调用工具 → 执行工具 → 获取结果 → 继续思考，直到任务完成。

**关键点**：没有额外的分类器或路由层，大模型自己决定一切。

---

## 工具调用 (Tool Call)

大模型通过输出特殊格式（如 JSON）来请求 Claude Code 执行特定操作（如读取文件、写入文件）。大模型不直接执行工具，只发出请求。

**格式示例**：`{"tool": "read", "file_path": "novel/chapter-03.md"}`

---

## 上下文窗口 (Context Window)

大模型一次能"看到"的所有内容的总量，包括用户输入、历史消息、工具返回结果等。Claude 的上下文窗口为 200K token（约 15 万字）。

**注意**：上下文窗口很大，但工具有独立的返回限制（如 Read 工具最多 2000 行）。

---

## 少脚手架，多模型 (Less Scaffolding, More Model)

Claude Code 的设计哲学：提供最少的预设规则和路由逻辑，让大模型自己决定如何使用工具。这与其他 AI Agent 形成对比，后者 often 有复杂的分类器、路由层和状态机。

---

## grep

一种文本搜索工具，用于在文件中查找特定内容。Claude Code 用它在大文件中精准定位需要的部分，而不是读取整个文件。

---

## 信息溢出 (Context Overflow)

当上下文窗口装满时，最早的内容会被主动丢弃。不是"放不下"，而是系统主动把旧内容推走，给新内容腾空间。

**关键点**：被丢弃的内容不是消失了，而是大模型看不见了。

---

## Lost in the Middle（中间遗失）

大模型对上下文窗口中**开头和结尾**的内容关注度更高，**中间**的内容容易被忽略。这是大模型注意力机制的固有特性。

**类比**：就像读一篇很长的文章，开头和结尾记得清楚，中间部分容易模糊。

---

## 系统提示 (System Prompt)

Claude Code 在每次对话开始时自动注入的指令，包括 CLAUDE.md 的内容。系统提示永远在上下文窗口的开头，不会被丢弃。

**关键点**：这就是为什么 CLAUDE.md 是存放关键设定的最佳位置。
