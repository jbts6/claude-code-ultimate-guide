核心循环：Claude Code 是怎么工作的

Lesson 001 · Claude Code · 2026-06-15

Part of your mission to 理解 Claude Code 的核心架构和工具体系，能够自主设计、调试和优化自己的 skill 工作流。

学完这节课，你将能够 解释 Claude Code 的核心交互循环，并理解它和其他 AI Agent 的本质区别。

例子 1：普通 AI 对话

当你在网页版 ChatGPT 或 Claude.ai 输入"帮我写一段对话"时，流程是这样的：

你 → AI 平台（加上系统提示词）→ 大模型 → 返回结果 → 展示给你

一次往返，结束。大模型只做一件事：接收你的输入，生成文字，返回给你。

例子 2：Claude Code 的做法

当你输入"帮我修改第 3 章的对话"时，Claude Code 做了完全不同的事情：

你说："帮我修改第 3 章的对话"
        ↓
Claude Code → 大模型
        ↓
大模型说："我需要先看看你的小说文件"
        ↓
Claude Code 执行 Read 工具，读取文件
        ↓
结果返回给大模型
        ↓
大模型说："我看到第 3 章的对话风格了，现在我来写"
        ↓
Claude Code 执行 Write 工具，写入文件
        ↓
大模型说："完成了，你要看看吗？"
        ↓
展示给你

你看到的一次交互，背后可能是 5 次、10 次、甚至 20 次的工具调用。

工具调用的机制

大模型本身只是一个文本生成器，它不能直接执行命令。那它是怎么调用工具的呢？

答案是：大模型输出一种特殊格式，Claude Code 能识别并执行。

我需要先读取第 3 章的内容。

{"tool": "read", "file_path": "novel/chapter-03.md"}

注意这两部分：

  • 普通文字："我需要先读取第 3 章的内容" — 这是给人看的
  • 特殊格式：{"tool": "read", ...} — 这是给 Claude Code 看的

Claude Code 看到这个 JSON 后，就知道模型想调用 read 工具，然后去执行，把结果返回给大模型。

工具的实际限制

即使上下文窗口很大（200K token），工具有自己的限制：

限制类型 是什么 大小
上下文窗口 大模型一次能"看到"的所有内容总和 200K token（约 15 万字）
工具返回限制 一次工具调用能返回多少内容 Read 工具：最多 2000 行

所以当文件很大时，大模型的策略是：先用 grep 找到关键部分的位置，再用 Read 工具只读取需要的部分。

检查你的理解

  1. Claude Code 和普通 AI 对话的核心区别是什么？
    答案

    普通对话是一问一答，一次往返就结束。Claude Code 运行一个"主循环"，大模型可以多次调用工具，直到任务完成。你看到的一次交互，背后可能有很多轮工具调用。

  2. 当大模型"调用工具"时，它实际上在做什么？
    答案

    大模型输出一段特殊格式的文本（如 JSON），告诉 Claude Code"我想用这个工具，参数是什么"。Claude Code 解析这个请求并执行，然后把结果返回给大模型。

  3. 为什么大模型选择用 grep 而不是直接读取整个大文件？
    答案

    两个原因：1) 工具有返回限制（如 Read 工具最多 2000 行）；2) 用 grep 精准定位再读取，比读取整个文件更节省 token 和上下文空间。

  4. 在主循环中，"大模型自己决定一切"是什么意思？没有哪些"其他程序"在做决定？
    答案

    没有额外的分类器告诉它"这是写代码任务"，没有路由层决定"这个请求应该发给哪个处理程序"。大模型读了文件内容后，自己判断该怎么改，然后自己决定调用什么工具。
