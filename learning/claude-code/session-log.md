## Session 2: 上下文管理

**日期**: 2026-06-15
**时长**: ~20 分钟
**主题**: 200K token 的秘密

### 学习目标
理解上下文窗口的工作原理，以及为什么 Claude Code 有时会"忘记"之前的信息

### 进度记录
- [x] 复习 Session 1（主循环、工具返回限制、大模型决策权）
- [x] 上下文窗口概念（桌子比喻）
- [x] 工具返回内容进入上下文（关键纠偏）
- [x] 信息溢出机制
- [x] Lost in the Middle 现象
- [x] 解决方案（CLAUDE.md、记忆工具）
- [x] 费曼检验

### 学习者表现
- 正确预测 A（纯对话）最快填满上下文
- 误解：以为工具读取的内容不进入上下文窗口 → 已纠正
- 误解：以为"每次重复"无效（实际是有效的，只是累）→ 已纠正
- 正确识别 CLAUDE.md 和记忆工具的有效性
- 费曼解释抓住核心，用"人的思维"类比很直观
- 自信度评分：8/10

### 关键理解
1. **上下文窗口是共享空间**：用户消息、模型回复、工具返回结果全部混在一起
2. **信息溢出**：上下文满了之后，最早的内容被主动丢弃
3. **Lost in the Middle**：模型对开头和结尾关注度更高，中间容易忽略
4. **工具返回进入上下文**：Read、Bash 等工具的输出都会进入上下文窗口
5. **CLAUDE.md 最可靠**：自动注入、永远在开头、不占对话空间

### 纠偏记录
- 学习者将"Claude"和"Claude Code"视为同一概念（Cloud = Claude Code 整体工具）
- 学习者以为工具读取的内容不进入上下文 → 实际上所有工具返回都进入上下文

### 下次复习
- 下次上课前复习上下文管理核心概念
- 重点：工具返回进入上下文、CLAUDE.md 的注入机制
